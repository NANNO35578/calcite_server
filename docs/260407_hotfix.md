# 文件管理模块 Hotfix 记录

**日期：** 2026年4月7日  
**问题：** 文件上传时 SQL 插入语句列数不匹配  
**修复者：** Kimi Code

---

## 一、问题描述

### 错误日志
```
20260407 16:55:55.131387 UTC 386892 ERROR error:1 status:1 - MysqlConnection.cc:254
20260407 16:55:55.131447 UTC 386892 ERROR Error(1136) [21S01] "Column count doesn't match value count at row 1" - MysqlConnection.cc:528
20260407 16:55:55.131478 UTC 386892 ERROR sql:insert into file_resource (id,user_id,file_name,file_type,file_size,object_key,status,created_at) values (default,1,'amethyst_sword.jpg','image/jpeg',1274,'1/2026/04/08/69d5371b82.jpg','processing',default,default) - MysqlConnection.cc:530
```

### 问题分析
SQL 语句中：
- **Columns**: `id, user_id, file_name, file_type, file_size, object_key, status, created_at` = **8列**
- **Values**: `default, 1, 'amethyst_sword.jpg', 'image/jpeg', 1274, '1/2026/04/08/69d5371b82.jpg', 'processing', default, default` = **9个值**

**原因：** values 部分多了一个 `default` 值

---

## 二、Root Cause

### 问题代码位置
文件：`./calcite/models/FileResource.h`  
函数：`sqlForInserting()`

### 问题根源
`sqlForInserting()` 函数中构建 INSERT 语句时，**columns 部分**和 **values 部分**的字段添加逻辑不一致：

**Columns 构建逻辑：**
```cpp
// updated_at (index 11) - 条件添加
if (dirtyFlag_[11]) {
    sql += "updated_at,";  // 只有当 dirtyFlag_[11] 为 true 时才添加
    ++parametersCount;
}
// created_at (index 10) - 总是添加
sql += "created_at,";
++parametersCount;
```

**Values 构建逻辑（修复前）：**
```cpp
// updated_at (index 11)
if (dirtyFlag_[11]) {
    sql.append("?,");
} else {
    sql += "default,";  // ❌ 错误：即使 columns 中没有 updated_at，values 也添加了 default
}
// created_at (index 10)
if (dirtyFlag_[10]) {
    sql.append("?,");
} else {
    sql += "default,";
}
```

**问题：**
- 当 `dirtyFlag_[11]` 为 `false` 时，`updated_at` **不会**被添加到 columns 中
- 但 values 部分由于 `else` 分支，仍然添加了 `default` 值
- 导致 columns 和 values 数量不匹配

---

## 三、修复方案

### 修复代码
```cpp
// status (index 9) - always in columns
if (dirtyFlag_[9]) {
    sql.append("?,");
} else {
    sql += "default,";
}
// updated_at (index 11) - conditional in columns
if (dirtyFlag_[11]) {
    sql.append("?,");
}
// ❌ 删除 else 分支，不再添加 default

// created_at (index 10) - always in columns
if (dirtyFlag_[10]) {
    sql.append("?,");
} else {
    sql += "default,";
}
```

### 修复原则
- **columns 中条件添加的字段，values 中也必须条件添加**
- 只有当字段确定会出现在 columns 中时，才在 values 中添加对应的值或 default
- 保持 columns 和 values 的字段顺序完全一致

```sql

VALUES (default,1,...,'processing',?,default);  -- 9列9值，匹配
```

## 六、预防措施

1. **代码审查**：ORM 模型的 `sqlForInserting()` 函数需要特别注意 columns 和 values 的同步
2. **单元测试**：添加针对插入操作的单元测试，覆盖各种字段组合
3. **规范**：
   - 条件字段（dirtyFlag 控制）在 columns 和 values 中必须使用相同的条件
   - 固定字段（总是添加）在 values 中必须总有对应值（`?` 或 `default`）
