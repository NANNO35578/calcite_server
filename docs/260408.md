# OCR功能实现文档

**日期：** 2026-04-08

**功能：** 实现OCR图片转文本接口，支持上传图片/PDF文件进行文字识别并自动创建笔记

---

## 一、新增接口

### 1. POST /api/ocr/recognize
上传文件进行OCR识别，异步处理流程：
1. 接收文件并创建 file_resource 记录（status = processing）
2. 调用第三方OCR API进行文字识别
3. 根据识别结果自动创建笔记
4. 上传原始文件到 MinIO
5. 更新 file_resource 状态为 done 或 failed

### 2. GET /api/ocr/status
查询OCR处理状态，若已完成则响应包含 note_id

## 四、关键代码说明

### 1. OCR API 配置
```cpp
static constexpr const char* API_URL = "https://jf3187r8t1fdu6r1.aistudio-app.com/layout-parsing";
static constexpr const char* API_TOKEN = "3ba24aebe8939b41b442baafcd198c3bf40975b5";
```

### 2. 请求JSON格式
```json
{
  "file": "base64encoded...",
  "fileType": 0,  // 0=PDF, 1=Image
  "useDocOrientationClassify": false,
  "useDocUnwarping": false,
  "useChartRecognition": false
}
```

### 3. 响应解析
从 `result.layoutParsingResults[].markdown.text` 提取文本内容

### 4. 笔记创建逻辑
- 标题：格式为 `ocr_YYYYMMDD_HHMMSS`（如 ocr_20260408_143052）
- 内容：OCR识别出的markdown文本（已清洗和转义）
- 摘要：不填写（避免中文字符截断问题）
- 文件夹：默认未分类（folder_id=0）

### 5. 文本清洗逻辑
在插入数据库前，对OCR返回的markdown文本进行清洗：
- **移除 null 字节** (`\0`)
- **保留正常空白字符**：换行(`\n`)、回车(`\r`)、制表符(`\t`)
- **转义其他控制字符**：将 `\x01-\x1F`（除`\t`,`\n`,`\r`外）转义为 `\xXX` 格式
- **处理 UTF-8 序列**：验证多字节UTF-8序列的合法性，丢弃无效序列

## 六、状态流转

```
processing ──▶ done  (OCR成功，笔记创建，MinIO上传成功)
processing ──▶ failed (OCR失败或后续处理失败)
```

---

## 七、注意事项

1. **异步处理**：OCR识别是异步的，接口立即返回 processing 状态
2. **轮询查询**：前端需轮询 `/api/ocr/status` 查询处理结果
3. **权限验证**：只能查询自己上传的文件状态
4. **超时设置**：OCR API 超时时间为300秒
5. **SSL验证**：OCR API调用暂时禁用SSL验证（生产环境需启用）
