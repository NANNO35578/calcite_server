## 二、实现内容

### 2.1 创建 ES 客户端类

**文件位置**:
- `calcite/utils/EsClient.h`
- `calcite/utils/EsClient.cc`

**功能特性**:
- `indexDocument()` - 异步索引文档（创建/替换）
- `updateDocument()` - 异步部分更新文档
- `deleteDocument()` - 异步删除文档
- `search()` - 异步全文搜索（支持高亮、分页）
- `searchSync()` - 同步搜索（用于简单场景）
- `ping()` - 检查 ES 连通性
- `createIndex()` - 创建索引并定义映射

**ES 索引映射**:
```json
{
  "mappings": {
    "properties": {
      "user_id": {"type": "long"},
      "title": {"type": "text", "analyzer": "standard"},
      "content": {"type": "text", "analyzer": "standard"},
      "summary": {"type": "text", "analyzer": "standard"},
      "tags": {"type": "keyword"},
      "updated_at": {"type": "date"}
    }
  }
}
```

**搜索配置**:
- 多字段匹配：title(权重3)、tags(权重2)、summary(权重2)、content(权重1)
- 高亮标签：`<mark>` 标签包裹匹配词
- 模糊匹配：支持 AUTO 级别的 fuzziness
- 用户过滤：通过 `user_id` 字段确保数据隔离
- 排序：按 `_score` 降序，再按 `updated_at` 降序

**创建笔记 (createNote)**:
1. 向 MariaDB 插入笔记记录
2. 在插入成功的回调中，异步向 ES 发起索引请求
3. 文档 ID 与数据库主键保持一致
4. 新笔记标签列表为空

```cpp

  note,
  [this, callback, userId](const drogon_model::calcite::Note &insertedNote) {
    int64_t noteId = insertedNote.getValueOfId();
    // 异步索引到ES
    indexNoteToES(noteId, userId, insertedNote, {});
    // ...
  }
);
```

**更新笔记 (updateNote)**:
1. 先更新 MariaDB 中对应记录
2. 在更新成功的回调中，异步向 ES 发起部分更新请求
3. 使用 `_update` API 只更新变更的字段

```cpp
noteMapper.update(
  updatedNote,
  [this, noteId, title, content, summary](size_t count) {
    // 异步更新ES
    esClient_.updateDocument(noteId, 
      title.empty() ? nullptr : &title,
      content.empty() ? nullptr : &content,
      summary.empty() ? nullptr : &summary,
      nullptr);
    // ...
  }
);
```

**删除笔记 (deleteNote)**:
1. 从 MariaDB 标记删除（软删除，设置 `is_deleted = 1`）
2. 在删除成功的回调中，异步向 ES 发起删除请求

```cpp
noteMapper.update(
  deletedNote,  // is_deleted = true
  [this, noteId](size_t count) {
    // 异步从ES删除
    esClient_.deleteDocument(noteId);
    // ...
  }
);
```

## 三、技术要点

### 3.1 异步设计

所有 ES 操作均采用异步方式：
- 不阻塞主业务逻辑
- 使用回调处理结果
- 错误仅记录日志，不影响用户响应

### 3.2 数据一致性

- **强一致性**: 数据库操作成功后才触发 ES 操作
- **最终一致性**: ES 同步存在延迟（通常毫秒级）
- **容错处理**: ES 失败不影响数据库操作

### 3.3 安全性

- 用户隔离：ES 查询中强制添加 `user_id` 过滤
- 软删除：数据库标记删除后从 ES 移除，防止搜索到已删除笔记

### 3.4 性能优化

- 批量查询：搜索时从 MariaDB 使用 `IN` 批量查询
- 分页支持：ES 原生分页，避免大数据量返回
- 字段裁剪：搜索结果不包含完整 `content`，减少网络传输
