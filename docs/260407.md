## 一、任务概述

为 `./calcite/` 项目实现一套完整的文件管理模块，支持文件上传、列表查询、删除、状态查询等功能，使用 MinIO 作为对象存储后端。

### API 清单

| 接口                | 方法   | 说明                    |
| ------------------ | ------ | ---------------------- |
| /api/file/upload   | POST   | 上传文件到 MinIO        |
| /api/file/list     | GET    | 获取文件列表             |
| /api/file/delete   | POST   | 删除文件（MinIO+数据库） |
| /api/file/status   | GET    | 查询文件上传状态          |
| /api/file/info     | GET    | 获取单个文件详情          |

## 三、数据库表结构

```sql
CREATE TABLE file_resource (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    note_id BIGINT,
    file_name VARCHAR(255) NOT NULL,
    file_path VARCHAR(512),
    file_type VARCHAR(50),
    file_size BIGINT,
    object_key VARCHAR(255) NOT NULL,
    url VARCHAR(512),
    status ENUM('processing', 'done', 'failed') DEFAULT 'processing',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES user(id),
    FOREIGN KEY (note_id) REFERENCES note(id),
    INDEX idx_user_id (user_id),
    INDEX idx_note_id (note_id),
    INDEX idx_status (status)
);
```

## 五、核心流程

### 5.1 文件上传流程

```
┌─────────┐    multipart/form-data     ┌──────────┐
│  Client │ ─────────────────────────▶ │  Drogon  │
└─────────┘                            └────┬─────┘
                                            │
                                            ▼
                                    ┌───────────────┐
                                    │ 1. 保存上传文件 │
                                    │ 2. 创建DB记录   │
                                    │    status=proc  │
                                    │ 3. 返回 file_id │
                                    └───────┬───────┘
                                            │
                                            ▼ (异步)
                                    ┌───────────────┐
                                    │ 4. 上传MinIO   │
                                    │ 5. 更新DB状态   │
                                    │    status=done │
                                    └───────────────┘
```

### 5.2 文件删除流程

```
Client ──▶ 验证权限 ──▶ 删除MinIO对象 ──▶ 删除DB记录 ──▶ 返回结果
```

### 5.3 权限控制

- 所有接口必须通过 JWT Token 鉴权
- 只能操作自己上传的文件（user_id 校验）
- 尝试访问/删除他人文件返回 `无权访问此文件` 错误

## 九、注意事项

1. **MinIO 服务**：确保 MinIO 服务已启动并可访问
2. **Bucket 创建**：首次使用前需在 MinIO 中创建 `notes-files` bucket
3. **异步上传**：上传接口立即返回，后台异步完成 MinIO 传输
4. **状态轮询**：客户端可通过 `/api/file/status` 轮询检查上传状态
5. **大文件处理**：当前实现适合中小文件（<100MB），超大文件建议使用分片上传
